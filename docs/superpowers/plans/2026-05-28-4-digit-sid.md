# 4-digit `SRII` sid Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Widen `sid` from a flat 2-digit value (1–255) to a 4-digit `site·role·index` (`SRII`) encoding, redesigning the MAC and TCP-port derivations that currently can't hold a 4-digit value.

**Architecture:** `sid = site×1000 + role×100 + index`. MACs split `sid` via `divmod(sid, 100)` into octet 4 (`sid_hi` = site·role) and octet 6 (`sid_lo` = index), with the per-interface index in octet 5. Ports become additive offsets (`telnet = 10000 + sid`, `monitor = 20000 + sid`) exposed as `SwitchConfig` properties. Validation widens to `1000–9999`.

**Tech Stack:** Python 3.13, PyYAML, dataclasses. No pytest suite (per `CLAUDE.md`); verification is via `python3 -c` behavioral one-liners, `--dry-run`, `mypy`, and `flake8` (169-char lines).

**Design spec:** `docs/superpowers/specs/2026-05-28-4-digit-sid-design.md`

**Working directory for all commands:** `config/nexus9000v/`

## Notes for the implementer

- **No pytest exists** — do not attempt `pytest`. Behavioral checks are the `python3 -c` one-liners and `--dry-run` above.
- **Do not touch** `ssh_*` helpers, `config/ansible/dynamic_inventory.py`, or `env_prod/` — all out of scope (the last must never be staged).
- The SMBIOS serial (`f"00000000{config.sid}"`) is intentionally left as-is; it lengthens naturally and is a known, accepted side effect (see spec).
- Validation is a **plain numeric range** by design — do not add role-digit validation.
